# Larsen & Toubro (L&T) — Industry 4.0 and IoT Training

*jun 2025 · on-site*

This was a short Industry 4.0 and IoT training program at L&T where we worked on small engineering-focused projects around automation, sensors, and embedded systems.

Most of the week was hands-on. We spent time understanding how industrial IoT systems are structured, how sensors communicate with controllers, and how real-time data pipelines are used in automation workflows. A lot of it was less about writing huge amounts of code and more about understanding how systems interact physically and digitally together.

For the final project, my team built a small ML-based home automation system focused on power optimization using live sensor inputs.

The basic idea was:
- collect real-time sensor data
- stream it through microcontrollers
- process the data centrally
- make automation decisions based on usage patterns and environmental conditions

We built communication pipelines between the sensor layer and the inference layer so data could move continuously instead of manually-triggered updates.

Looking back, the project itself wasn't some massive production system or anything — but it was one of the first times I started thinking more seriously about systems beyond just writing isolated code. Things like:
- data flow
- hardware constraints
- real-time communication
- reliability
- sensor noise
- latency between devices

started becoming practical engineering problems instead of theoretical concepts.

The project ended up receiving the **Best Project Award** among 50+ trainees in the program.

What I probably took away most from that week was realizing how much modern engineering becomes about connecting systems together properly rather than just building one isolated component well.
